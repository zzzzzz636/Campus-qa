# 系统 V1 运行说明

## 1. 项目名称

校园生活百事通智能问答知识库系统。

## 2. 第二周 V1 已实现功能

- 后端连接 SQLite 数据库。
- FAQ 列表查询。
- 问答检索从数据库 `faq` 表返回答案。
- 查询成功或失败都会写入 `query_log` 查询日志表。
- 查询成功后会更新对应 FAQ 的 `view_count`。
- 用户贡献问答提交入库，保存到 `contribution` 表。
- 前端页面已调用后端接口展示结果：
  - `result.html` 调用 `/api/qa/search`
  - `contribute.html` 调用 `/api/contribution/add`
  - `admin.html` 调用 `/api/faq/list`
- 前端页面已同步到 Spring Boot 静态资源目录，可通过 `http://localhost:8080/` 同源访问。

## 3. 当前未实现功能

- 管理员真实登录鉴权。
- FAQ 新增、修改、删除完整 CRUD。
- 贡献审核通过/拒绝入库。
- 知识导入。
- 大模型问答。
- 官网爬虫。

## 4. 运行环境

当前本机验证环境：

```text
JDK: OpenJDK 25.0.1
Maven: Apache Maven 3.9.16
Spring Boot: 3.3.6
数据库: SQLite
SQLite JDBC: 3.46.1.0
```

项目 `pom.xml` 中配置的 Java release 为：

```text
Java 17
```

因此后续在其他机器运行时，建议使用 JDK 17 或更高版本。

## 5. 数据库初始化步骤

数据库配置在：

```text
backend/src/main/resources/application.yml
```

数据库文件路径：

```text
database/campusqa.db
```

后端从 `backend` 目录启动时，配置使用：

```text
jdbc:sqlite:../database/campusqa.db
```

初始化脚本：

```text
database/schema.sql
database/sample-data.sql
```

当前项目已关闭 Spring Boot 自动执行 SQL：

```yml
spring.sql.init.mode: never
```

因此首次运行前需要手动初始化数据库：

```powershell
cd E:\code\shixun\campus-qa
python database\init_sqlite.py
```

该脚本会执行：

```text
database/schema.sql
database/sample-data.sql
```

如果 `database/campusqa.db` 已存在，脚本会补齐表结构和初始数据，不会像第一周版本那样每次启动删除所有数据。

注意：

- `database/campusqa.db` 是本地运行生成文件，不建议提交到 Git。
- 项目 `.gitignore` 已忽略 `*.db`、`*.db-shm`、`*.db-wal`。
- 当前配置不会在每次启动时重建数据库，因此用户贡献、查询日志和 FAQ 浏览次数可以在重启后保留。

## 6. 后端启动步骤

进入后端目录：

```powershell
cd E:\code\shixun\campus-qa\backend
```

使用项目本地 Maven 启动：

```powershell
..\..\.tools\apache-maven-3.9.16\bin\mvn.cmd spring-boot:run
```

如果 Maven 已加入系统 PATH，也可以使用：

```powershell
mvn spring-boot:run
```

启动成功后，后端地址为：

```text
http://localhost:8080
```

健康检查：

```text
http://localhost:8080/api/hello
```

## 7. 前端访问地址

推荐通过 Spring Boot 同源访问：

```text
http://localhost:8080/
http://localhost:8080/index.html
http://localhost:8080/result.html?q=图书馆
http://localhost:8080/contribute.html
http://localhost:8080/admin.html
http://localhost:8080/login.html
```

说明：

- `backend/src/main/resources/static/` 中保存了用于运行演示的前端页面。
- `frontend/` 目录中保留了原始静态页面文件。
- 不建议直接双击打开 HTML 文件测试接口，因为直接打开可能不是同源访问，`fetch('/api/...')` 可能无法正确请求后端。

## 8. 接口测试地址

健康检查：

```text
GET http://localhost:8080/api/hello
```

FAQ 列表：

```text
GET http://localhost:8080/api/faq/list
```

问答检索：

```text
GET http://localhost:8080/api/qa/search?question=图书馆
GET http://localhost:8080/api/qa/search?question=食堂
GET http://localhost:8080/api/qa/search?question=校园卡
```

用户贡献提交：

```text
POST http://localhost:8080/api/contribution/add
Content-Type: application/json
```

请求体示例：

```json
{
  "question": "宿舍水电费在哪里查询？",
  "suggestedAnswer": "可通过学校生活服务平台查询。",
  "category": "宿舍报修"
}
```

成功返回示例：

```json
{
  "success": true,
  "message": "提交成功，等待管理员审核",
  "id": 3
}
```

## 9. 常见问题处理

### 9.1 端口 8080 被占用

检查占用进程：

```powershell
Get-NetTCPConnection -LocalPort 8080 -State Listen
```

停止指定进程：

```powershell
Stop-Process -Id <进程号> -Force
```

### 9.2 页面提示后端服务不可用

处理方式：

- 确认 Spring Boot 已启动。
- 确认访问地址是 `http://localhost:8080/...`。
- 不要直接双击 HTML 文件测试接口。

### 9.3 FAQ 数据为空或数据库异常

处理方式：

- 确认 `database/schema.sql` 和 `database/sample-data.sql` 存在。
- 执行 `python database\init_sqlite.py` 初始化数据库。
- 如需完全重置演示数据，可以先删除本地生成的 `database/campusqa.db`，再执行初始化脚本。

### 9.4 Maven 命令不可用

本项目已下载本地 Maven：

```text
E:\code\shixun\.tools\apache-maven-3.9.16
```

可使用完整路径运行：

```powershell
E:\code\shixun\.tools\apache-maven-3.9.16\bin\mvn.cmd spring-boot:run
```

### 9.5 JDK 25 警告

当前使用 JDK 25 运行时可能出现 SQLite 或测试框架的 warning。当前验证中这些 warning 不影响 V1 功能运行。正式环境建议使用 JDK 17 或 21。
