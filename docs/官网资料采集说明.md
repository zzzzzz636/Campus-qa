# 官网资料采集说明

本项目第四周 V3 已将爬虫功能整理为后端接口和后台页面按钮，不再使用离线 Python 脚本生成 CSV。

## 使用方式

1. 编辑 `tools/seed_urls.txt`。
2. 每行填写一个公开网页 URL。
3. 启动 Spring Boot 后端。
4. 打开 `http://localhost:8080/admin.html`。
5. 在“官网资料采集”区域选择采集数量，点击“开始采集并入库”。

## 后端接口

```http
POST /api/crawler/run?limit=3
```

说明：

- 最多采集 3 条 URL。
- URL 来源为 `tools/seed_urls.txt`。
- 只支持公开 `http` 或 `https` 页面。
- 不访问登录页、统一认证、教务系统等页面。
- 采集成功后直接写入 `knowledge_doc` 表。
- 如果 URL 已经存在于知识资料库，会跳过并返回提示。

## 入库字段

采集成功后写入：

- `title`：网页标题
- `content`：网页正文文本
- `category`：校园资料
- `source_url`：原网页 URL
- `source_type`：官网爬虫

## 查询展示

采集成功后：

1. 后台“知识资料库”会自动刷新并显示新增资料。
2. 首页搜索相关关键词时，如果 FAQ 未命中，会继续从知识资料库匹配。
3. 结果页会显示“根据校园资料库匹配”。

## 当前限制

- 不做全站递归爬取。
- 不处理需要登录、验证码或动态渲染的页面。
- 正文提取为轻量实现，可能包含导航、页脚等噪声。
- 不支持复杂网页清洗、去重合并和自动分类。
